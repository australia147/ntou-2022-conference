# 📝 網站編輯指南

> 這份指南教你如何使用後台編輯網站內容。即使你不會寫程式，也能輕鬆修改文字、上傳圖片、發布更新。

## 🚀 快速開始

### 第一次使用（只需要做一次）

1. **註冊 GitHub 帳號**：到 [github.com](https://github.com) 註冊（免費）
2. **請網站擁有者把你加入協作者**：擁有者要在 GitHub repo 設定裡 invite 你
3. **接受邀請**：你的 email 會收到邀請信，點 Accept

### 進入後台

打開瀏覽器，輸入：

```
https://tpr2026.ntouctl.org/admin/
```

第一次進去會要你 **「Sign in with GitHub」**，點下去 → 授權 → 完成。

之後就會記住你的登入。

---

## 🎨 後台介面長什麼樣子？

進入後台後，左邊會看到 6 個分類：

| 分類 | 內容 |
|---|---|
| 🌊 **活動基本資訊** | 活動名稱、日期、地點、報名連結（所有頁面共用） |
| 🏠 **首頁內容** | 首頁的介紹、卡片、統計數字、報名 CTA |
| 📅 **議程** | 時間軸、講者、各場次 |
| 🧭 **場域踏查** | 兩條路線的站點、費用 |
| 🚌 **交通資訊** | 接駁車、自駕、大眾運輸 |
| 📞 **聯繫我們** | LINE@、承辦人 |

點任一個分類，就能看到表單，直接填字。

---

## ✏️ 怎麼編輯內容？

### 例：改活動日期

1. 點左邊 🌊 **活動基本資訊**
2. 點 **活動資訊**
3. 找到 **「活動日期」** 欄位，改成新的日期
4. 點右上角 **「Save」**（儲存）
5. 點 **「Publish」→「Publish now」**（發布）
6. 等約 1 分鐘，網站自動更新 ✅

### 例：新增一個議程時段

1. 點 📅 **議程** → 點 **議程內容**
2. 滾到 **「議程時段」** 區塊
3. 點 **「Add 議程時段」** 按鈕
4. 填寫時間、標題、地點等
5. 用拖曳調整時段順序
6. 儲存 → 發布

## 📷 在內容裡放圖片（重要更新！）

從 2026/05 起，**幾乎所有「敘述類」欄位**都升級成富文字編輯器（Rich Text），可以直接放圖。包括：

- 活動標語、單位全名
- 首頁介紹段落、卡片描述、報名 CTA 描述
- 議程的「副標題（補充說明）」、底部提醒卡片
- 場域踏查的路線介紹、站點說明、費用提醒
- 交通資訊的自駕說明、停車說明、大眾運輸說明
- LINE@ 客服說明等

### 怎麼判斷哪個欄位可以放圖？

進入欄位後，**上方會出現一條工具列**，長得像：

```
[B] [I] [H1] [H2] [清單] [連結] [📷 圖片] [引言] ...
```

有工具列的欄位 = 可以放圖、加粗、做清單。沒有工具列的就只是普通文字（例如「電話」、「日期」這種短欄位）。

### 步驟示範：在路線介紹放一張港口照片

1. 進入後台 → 點 🧭 **場域踏查**
2. 點 **踏查內容** → 找到 **「路線一」**
3. 點 **「路線介紹」** 欄位（會看到工具列）
4. 把游標移到你要插圖的位置
5. 點工具列的 **📷 圖片** 按鈕
6. 跳出視窗 → 點 **「Upload」** → 選擇電腦上的圖檔
7. 上傳完成，圖片就出現在文字裡 👏
8. 可以繼續打字、再插一張，或調整位置
9. 全部改完 → 右上角 **Save** → **Publish → Publish now**
10. 約 1 分鐘後網站更新

### 圖片小技巧

| 項目 | 建議 |
|---|---|
| **格式** | JPG / PNG / WebP 都可以 |
| **大小** | 寬度建議 **1200px 以內**，檔案 < 500KB |
| **檔名** | 用英文+數字（例：`harbor-2026.jpg`），中文檔名有時會出狀況 |
| **位置** | 圖片會自動存到 `assets/images/uploads/` 資料夾，你不用管 |
| **替代文字** | 上傳時可以填，給看不到圖的人聽，也對 SEO 好 |

### 我想用網路上的圖片（不上傳）

直接在文字編輯器點 📷 → 切到 **「URL」** 分頁 → 貼網址。

⚠️ 注意：外部圖片如果被刪掉，你的網頁也會跟著掉圖。**建議還是下載後上傳。**

## ⚠️ 注意事項

### ✅ 可以做的事

- 改任何文字（標題、描述、地點等）
- 上傳/換圖片
- 增加/刪除項目（議程、站點、承辦人等）
- 重新排序（用拖曳）

### ❌ 請不要做的事

- ❌ 直接改 GitHub 上的 `.html`、`.css`、`.js` 檔（會把網站弄壞）
- ❌ 把首頁卡片數量改成不是 **4 張**（會破版）
- ❌ 把首頁統計數字改成 **超過 4 個**（卡片 grid 最多 4 欄；目前是 3 個，剛好）
- ❌ 上傳 > 5MB 的大圖（網站會跑很慢）

### 🆘 不小心改壞了怎麼辦？

不用怕！每次儲存都會留紀錄，網站擁有者可以**還原到任何一個之前的版本**。

如果你發現網站壞掉，**先別繼續編輯**，直接聯絡擁有者。

## ✨ 文字排版：Markdown 語法速查

有工具列的欄位都支援 **Markdown**。最快的學法是直接點工具列按鈕，但如果你想用打字的：

| 想要的效果 | 怎麼打 | 顯示 |
|---|---|---|
| **粗體** | `**重點**` | **重點** |
| *斜體* | `*斜體*` | *斜體* |
| 大標題 | `# 標題` | 標題（很大） |
| 中標題 | `## 標題` | 標題（中） |
| 小標題 | `### 標題` | 標題（小） |
| 項目清單 | `- 第一項`<br>`- 第二項` | • 第一項<br>• 第二項 |
| 編號清單 | `1. 第一項`<br>`2. 第二項` | 1. 第一項<br>2. 第二項 |
| [連結](https://example.com) | `[連結文字](網址)` | 點下去會跳轉 |
| 插入圖片 | `![說明](圖片網址)` | 顯示圖片 |
| 引言區塊 | `> 這是引言` | 左側帶藍線的區塊 |
| 換行 | 直接按 Enter | 自動換行 |
| 空白行（分段） | 按兩次 Enter | 段落間多一點距離 |

💡 **不用全背**。最常用的就 5 個：**粗體、清單、連結、圖片、標題**，這些工具列都有按鈕。

## 🤖 用 AI 幫你把純文字改成 Markdown 格式

如果你手邊有一段 Word 或 email 裡的純文字，想快速貼進後台又有漂亮排版，可以直接交給 AI（ChatGPT、Claude、Gemini 都可以）幫你轉。

### 萬用提示詞（複製貼上就能用）

```
請幫我把下面這段文字轉成 Markdown 格式，準備貼到網站後台。要求：
1. 把重點字加粗（**這樣**）
2. 條列項目用「- 」開頭
3. 有編號的步驟用「1. 2. 3.」
4. 如果有網址，幫我做成 [文字](網址) 的連結
5. 段落之間留一個空行
6. 不要加 #標題（因為標題另外有欄位）
7. 直接輸出 Markdown 原文，不要包在程式碼框裡

文字如下：
---
（在這裡貼你的文字）
---
```

### 例：給 AI 這段

```
這條路線從和平島出發，沿途會經過諸聖教堂遺址、張火焰故事館、海遊學堂、和平島市場，最後返回海洋大學。重點是：歷史走讀、食魚教育、秘境探索。如果想看詳細地圖，可到 https://example.com 查看。
```

### AI 會回你這樣

```
這條路線從 **和平島** 出發，沿途會經過：

- 諸聖教堂遺址
- 張火焰故事館
- 海遊學堂
- 和平島市場

最後返回海洋大學。重點是：**歷史走讀**、**食魚教育**、**秘境探索**。

如果想看詳細地圖，可到 [這個頁面](https://example.com) 查看。
```

把這段直接貼進後台的「路線介紹」欄位 → 儲存 → 發布即可！

### 想連圖片位置都讓 AI 規劃

```
請幫我把下面這段文字轉成 Markdown，並在你覺得適合放圖片的地方
插入 `![請填圖片說明](待上傳)` 當佔位符，我會自己把圖片補上。

文字如下：
---
（你的文字）
---
```

AI 會幫你標好哪段適合配圖，你只要在後台對應位置把 `(待上傳)` 換成真正的圖就好。

### 進階：把整個 Word 檔改成多段

如果你的內容很長、有多個小節，可以這樣問：

```
我有一段活動說明，請幫我：
1. 拆成 3 段
2. 每段開頭加一個 ### 小標題
3. 每段裡的重點用粗體標出來
4. 在每段最後留一個 `![圖片建議：xxx](待上傳)` 註明適合放什麼樣的圖

文字如下：
（貼上）
```

## 🆘 常見問題

### Q: 我改完按 Save，但網站沒變？
A: Save 只是儲存草稿。要按 **Publish → Publish now** 才會真的發布。

### Q: 我登入時看到「You don't have access」？
A: 表示你還沒被加為協作者，請聯絡擁有者把你 invite 進 repo。

### Q: 我可以用手機編輯嗎？
A: 可以，但建議用電腦比較順手，特別是要上傳圖片時。

### Q: 我能看到別人改了什麼嗎？
A: 可以，到 [GitHub repo 的 Commits 頁](https://github.com/australia147/ntou-2022-conference/commits) 就能看完整修改紀錄。
