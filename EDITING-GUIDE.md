# 📝 網站編輯指南

> 這份指南教你如何使用後台編輯網站內容。即使你不會寫程式，也能輕鬆修改文字、上傳圖片、發布更新。

## 🚀 快速開始

### 第一次使用（只需要做一次）

1. **註冊 GitHub 帳號**：到 [github.com](https://github.com) 註冊（免費）
2. **請網站擁有者把你加入協作者**：擁有者要在 GitHub repo 設定裡 invite 你
3. **接受邀請**：你的 email 會收到邀請信，點 Accept

### 進入後台

打開瀏覽器，輸入：

```
https://ntou2022.ntouctl.org/admin/
```

第一次進去會要你 **「Sign in with GitHub」**，點下去 → 授權 → 完成。

之後就會記住你的登入。

---

## 🎨 後台介面長什麼樣子？

進入後台後，左邊會看到 6 個分類：

| 分類 | 內容 |
|---|---|
| 🌊 **活動基本資訊** | 活動名稱、日期、地點、報名連結（所有頁面共用） |
| 🏠 **首頁內容** | 首頁的介紹、卡片、統計數字、報名 CTA |
| 📅 **議程** | 時間軸、講者、各場次 |
| 🧭 **場域踏查** | 兩條路線的站點、費用 |
| 🚌 **交通資訊** | 接駁車、自駕、大眾運輸 |
| 📞 **聯繫我們** | LINE@、承辦人 |

點任一個分類，就能看到表單，直接填字。

---

## ✏️ 怎麼編輯內容？

### 例：改活動日期

1. 點左邊 🌊 **活動基本資訊**
2. 點 **活動資訊**
3. 找到 **「活動日期」** 欄位，改成新的日期
4. 點右上角 **「Save」**（儲存）
5. 點 **「Publish」→「Publish now」**（發布）
6. 等約 1 分鐘，網站自動更新 ✅

### 例：新增一個議程時段

1. 點 📅 **議程** → 點 **議程內容**
2. 滾到 **「議程時段」** 區塊
3. 點 **「Add 議程時段」** 按鈕
4. 填寫時間、標題、地點等
5. 用拖曳調整時段順序
6. 儲存 → 發布

### 例：上傳一張圖片（如講者照片）

1. 在編輯任何地方看到 **圖片欄位** 時，點 **「Upload」**
2. 選擇檔案
3. 系統會自動把圖片放到正確的位置
4. 儲存 → 發布

> 💡 圖片建議格式：JPG 或 PNG，寬度建議 800px 以下（太大會讓網站變慢）

## 🎯 常用編輯場景

### 改報名連結

🌊 活動基本資訊 → 報名表單網址 → 貼上 Google Form 完整網址（含 `https://`）

### 加新承辦人

📞 聯繫我們 → 承辦人列表 → Add → 填姓名/電話/分機

### 改議程的某個 A/B/C 場次內容

📅 議程 → 找到對應時段 → 展開「平行場次群組」→ 找到場次 → 改

### 改場域踏查的站點

🧭 場域踏查 → 路線列表 → 展開「行程站點」→ 編輯/新增/刪除

## ⚠️ 注意事項

### ✅ 可以做的事

- 改任何文字（標題、描述、地點等）
- 上傳/換圖片
- 增加/刪除項目（議程、站點、承辦人等）
- 重新排序（用拖曳）

### ❌ 請不要做的事

- ❌ 直接改 GitHub 上的 `.html`、`.css`、`.js` 檔（會把網站弄壞）
- ❌ 把首頁卡片數量改成不是 4 張（會破版）
- ❌ 把首頁統計數字改成不是 4 個

### 🆘 不小心改壞了怎麼辦？

不用怕！每次儲存都會留紀錄，網站擁有者可以**還原到任何一個之前的版本**。

如果你發現網站壞掉，**先別繼續編輯**，直接聯絡擁有者。

## 💡 文字格式小技巧

某些欄位（提示寫「可用 ** ** 表示粗體」的）支援簡單格式：

| 你輸入 | 顯示效果 |
|---|---|
| `這是 **重點** 文字` | 這是 **重點** 文字 |
| `第一行<br>第二行` | 第一行（換行）第二行 |
| 直接按 Enter 換行 | 也會換行 |

## 🔄 修改後多久會生效？

```
你按 Publish
   ↓
GitHub 記錄修改（1 秒）
   ↓
GitHub Pages 自動部署（30–60 秒）
   ↓
網站更新完成 ✅
```

通常 **1 分鐘內** 就能看到。如果超過 5 分鐘還沒更新，按 **Ctrl + F5** 強制重新整理瀏覽器。

## 🆘 常見問題

### Q: 我改完按 Save，但網站沒變？
A: Save 只是儲存草稿。要按 **Publish → Publish now** 才會真的發布。

### Q: 我登入時看到「You don't have access」？
A: 表示你還沒被加為協作者，請聯絡擁有者把你 invite 進 repo。

### Q: 我可以用手機編輯嗎？
A: 可以，但建議用電腦比較順手，特別是要上傳圖片時。

### Q: 我能看到別人改了什麼嗎？
A: 可以，到 [GitHub repo 的 Commits 頁](https://github.com/australia147/ntou-2022-conference/commits) 就能看完整修改紀錄。
